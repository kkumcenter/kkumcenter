# Supabase 실행용 SQL 안내

이 폴더는 군북면 꿈키움센터 홈페이지를 실제 기능형 사이트로 전환하기 위한 Supabase 실행용 SQL 파일을 모아둔 곳입니다.

## 현재 단계

현재는 `실행용 SQL 파일 작성`과 `비회원 신청 서버 함수 준비` 단계까지 완료했습니다.

진행 순서는 다음과 같습니다.

1. 운영 기준 정리: 완료
2. 관리자 페이지 기획서 1차안 작성: 완료
3. Supabase DB 구조 설계: 완료
4. Supabase SQL 설계안 작성: 완료
5. 실행용 SQL 파일 작성: 완료
6. 비회원 신청 서버 함수 작성: 완료
7. Supabase 프로젝트 생성: 다음 단계
8. Supabase SQL Editor에서 SQL 실행: 다음 단계
9. Supabase Edge Function 배포: 다음 단계
10. 관리자 페이지와 DB 연결 검증: 이후 단계

## 파일 실행 순서

Supabase 프로젝트를 만든 뒤 SQL Editor에서 아래 순서대로 실행합니다.

1. `schema.sql`
   - 테이블, 상태값, 기본 함수, 자동 수정일 처리 등을 만듭니다.

2. `seed.sql`
   - 임시 관리자 이메일 2개와 공간 6개 기본 데이터를 넣습니다.

3. `policies.sql`
   - 회원, 비회원, 관리자 권한 규칙과 파일 저장소 버킷을 설정합니다.

## 서버 함수

비회원 공간예약, 교육신청, 문의/제안은 아래 Edge Function에서 처리합니다.

- `functions/public-submit/index.ts`

이 함수는 비회원 조회 비밀번호를 암호화한 뒤 DB에 저장합니다.
교육신청은 정원을 확인해 `신청완료` 또는 `대기` 상태로 접수합니다.

## 중요한 운영 기준

- 공간예약은 중복 시간대도 일단 모두 접수합니다.
- 관리자가 관리자 화면에서 중복 가능 예약을 확인하고 승인 여부를 결정합니다.
- 교육신청은 선착순으로 운영합니다.
- 정원 초과와 대기 순번 처리는 `public-submit` Edge Function에서 처리합니다.
- 비회원 조회 비밀번호는 원문으로 저장하지 않고 암호화된 값만 저장해야 합니다.
- 공지사항과 마을 이야기 첨부파일은 공개 다운로드 기준입니다.
