# 군북면 꿈키움센터 관리자 페이지 기획서 1차안

## 1. 기획 목적

꿈키움센터 홈페이지는 단순 소개형 사이트가 아니라 공간예약, 교육신청, 문의/제안, 게시판, 갤러리 운영이 함께 이루어지는 공공·로컬 플랫폼이다.  
따라서 관리자 페이지는 기능을 나중에 덧붙이는 보조 화면이 아니라, 실제 운영 기준을 안정적으로 실행하는 중심 화면으로 설계한다.

관리자 페이지의 1차 목표는 다음과 같다.

- 운영자가 매일 확인해야 하는 신규 등록 건을 빠르게 파악한다.
- 공간예약, 교육신청, 문의/제안의 상태를 일관된 기준으로 관리한다.
- 공지사항, 마을 이야기, 갤러리 게시물을 안정적으로 관리한다.
- 회원과 비회원의 이용 흐름을 구분하되, 이용 문턱은 낮게 유지한다.
- 관리자 작업 기록과 숨김 처리 기준을 최소한으로 마련해 운영 안정성을 확보한다.

## 2. 확정된 운영 기준

### 기본 운영

- 실제 관리자는 2명이다.
- 관리자 권한은 1차 개발에서는 하나로 통일한다.
- 운영자가 매일 가장 먼저 확인할 항목은 신규 공간예약, 신규 교육신청, 신규 문의/제안이다.
- 초기 알림은 별도 문자나 카카오 알림 없이 관리자 화면에서 확인하는 방식으로 운영한다.
- 문자, 카카오 알림톡, 이메일 알림은 2차 개발에서 회원 대상으로 확장한다.

### 회원과 비회원

비회원은 신청 문턱을 낮추기 위한 간편 이용자이고, 회원은 반복 이용과 기록 관리가 편리한 이용자로 구분한다.

- 비회원도 이름과 연락처를 입력하면 공간예약, 교육신청, 문의/제안을 이용할 수 있다.
- 비회원은 알림 기능을 제공하지 않는다.
- 비회원은 신청 또는 문의 조회 시 이름, 연락처, 비밀번호 등의 확인 정보를 입력한다.
- 회원은 가입 후 바로 이용할 수 있으며 별도 관리자 승인은 받지 않는다.
- 회원 필수 정보는 이름, 연락처, 지역이다. 지역은 시·군·구까지만 입력한다.
- 회원 선택 정보는 이메일, 생년월일이다.
- 회원은 마이페이지에서 공간예약, 교육신청, 문의 답변을 확인할 수 있다.
- 회원은 마을 이야기와 꿈센터 갤러리를 작성할 수 있다.
- 회원만 2차 개발 알림 기능의 대상이 된다.

### 공간예약

- 예약 가능한 공간은 공유주방, 대회의실, 강의실, 소회의실, 청소년활동실, 미디어스튜디오 6개다.
- 공간예약은 관리자 승인제로 운영한다.
- 예약 가능 시간은 평일 09:00~18:00을 기본으로 한다.
- 주말과 공휴일 예약도 가능하다.
- 이용일 기준 7일 전에 신청하는 것을 권장하지만 당일에도 가능하다.
- 예약 취소는 당일에도 가능하다.
- 같은 시간대 중복 예약은 자동 차단보다 관리자 검토 후 승인하는 방식으로 시작한다.
- 예약 상태값은 접수, 승인, 반려, 취소로 관리한다.
- 반려 사유는 1차 개발에서는 별도 필수 입력으로 두지 않는다.
- 예약 신청 항목은 신청자명, 연락처, 예약일, 시간, 공간, 목적, 사용인원, 기타로 구성한다.

### 교육신청

- 교육신청은 선착순으로 운영한다.
- 정원이 차면 자동 마감한다.
- 정원 초과 시 대기자 접수를 받는다.
- 한 사람이 같은 프로그램을 중복 신청할 수 있다.
- 신청 취소는 가능하게 한다.
- 신청 상태값은 신청완료, 대기, 승인, 취소로 관리한다.
- 프로그램 등록 항목은 프로그램명, 기간, 장소, 정원, 대상, 강사, 내용, 신청기간, 이미지로 구성한다.
- 신청자 정보는 이름, 연락처, 출생연도, 지역으로 구성한다.
- 프로그램 종료 후 출석이나 수료 여부는 1차 개발에서 관리하지 않는다.

### 문의 / 제안

- 문의/제안은 회원과 비회원 모두 작성할 수 있다.
- 문의/제안은 비공개 1:1 문의로 운영한다.
- 답변은 관리자만 작성한다.
- 문의 처리 상태값은 접수, 확인중, 답변완료로 관리한다.
- 비회원 조회를 위해 비밀번호 확인 방식을 유지한다.
- 비회원은 작성자명, 연락처, 비밀번호로 답변을 조회한다.
- 회원은 로그인 후 마이페이지에서 내 문의와 답변을 확인한다.

### 게시판과 갤러리

- 공지사항은 관리자만 작성한다.
- 마을 이야기는 회원가입 시 작성할 수 있으며 작성 즉시 공개한다.
- 꿈센터 갤러리는 회원가입 시 작성할 수 있으며 여러 장 사진 업로드가 가능하다.
- 마을 이야기와 갤러리는 공공 목적에 부합하지 않는 경우 관리자가 삭제 또는 비공개 처리할 수 있다.
- 게시글 상태값은 공개, 비공개, 임시저장으로 관리한다.
- 게시글 첨부파일 기능은 필요하다.
- 갤러리 항목은 제목, 날짜, 장소, 설명, 대표사진, 추가사진으로 구성한다.
- 갤러리 공개 여부 선택과 행사별 앨범 기능은 1차 개발에서는 제외한다.

### 관리자 기록과 삭제 기준

- 관리자 작업 기록은 최소한으로 남긴다.
- 기록 대상은 예약 승인/반려/취소, 문의 답변, 게시글 비공개 처리, 게시글 삭제 처리다.
- 기록 항목은 작업자, 작업일시, 대상 종류, 대상 이름 또는 번호, 작업 내용으로 구성한다.
- 삭제는 완전 삭제보다 숨김 처리를 기본으로 한다.
- 숨김 처리된 데이터는 일반 이용자 화면에서는 보이지 않지만, 관리자 화면에서는 확인할 수 있게 한다.

## 3. 관리자 메뉴 구조

### 1차 관리자 메뉴

1. 대시보드
2. 공간예약 관리
3. 교육신청 관리
4. 프로그램 관리
5. 문의 / 제안 관리
6. 게시판 관리
7. 갤러리 관리
8. 회원 관리
9. 통계
10. 관리자 기록

### 대시보드

운영자가 로그인 후 가장 먼저 보는 화면이다.

표시 항목은 다음과 같다.

- 오늘 예약
- 이번 주 예약
- 신규 공간예약
- 신규 교육신청
- 미답변 문의
- 최근 게시글
- 최근 갤러리 등록

1차 개발에서는 숫자 요약 카드와 최근 목록 중심으로 구성한다.

### 공간예약 관리

공간예약 신청 건을 확인하고 상태를 변경하는 화면이다.

목록에서 확인할 정보는 다음과 같다.

- 예약번호
- 신청자명
- 연락처
- 공간명
- 예약일
- 예약시간
- 사용인원
- 상태
- 접수일

상세 화면에서는 목적, 기타, 신청자 유형, 관리자 작업 기록을 확인한다.

관리자가 할 수 있는 작업은 다음과 같다.

- 승인
- 반려
- 취소 처리
- 비공개 또는 숨김 처리

### 교육신청 관리

프로그램별 신청자를 확인하고 상태를 관리하는 화면이다.

목록에서 확인할 정보는 다음과 같다.

- 신청번호
- 프로그램명
- 신청자명
- 연락처
- 출생연도
- 지역
- 신청상태
- 신청일

관리자가 할 수 있는 작업은 다음과 같다.

- 신청완료 확인
- 대기자 확인
- 승인
- 취소 처리

정원 초과 시 신청자는 자동으로 대기 상태가 된다.

### 프로그램 관리

운영자가 프로그램을 등록하고 수정하는 화면이다.

프로그램 등록 항목은 다음과 같다.

- 프로그램명
- 대표 이미지
- 모집 상태
- 신청기간
- 운영기간
- 장소
- 정원
- 대상
- 강사
- 프로그램 내용

프로그램 상태는 접수예정, 접수중, 접수마감으로 관리한다.

### 문의 / 제안 관리

비공개 1:1 문의를 확인하고 답변하는 화면이다.

목록에서 확인할 정보는 다음과 같다.

- 문의번호
- 작성자
- 연락처
- 제목
- 처리상태
- 접수일
- 회원 여부

상세 화면에서는 문의 내용, 답변 작성란, 처리상태 변경 기능을 제공한다.

처리 상태는 접수, 확인중, 답변완료로 관리한다.

### 게시판 관리

공지사항과 마을 이야기를 관리한다.

공지사항은 관리자 등록, 수정, 비공개, 숨김 처리를 제공한다.

마을 이야기는 회원 작성글을 확인하고, 공공 목적에 부합하지 않는 경우 비공개 또는 숨김 처리한다.

공통 관리 항목은 다음과 같다.

- 제목
- 작성자
- 작성일
- 조회수
- 공개 상태
- 첨부파일

### 갤러리 관리

회원이 작성한 갤러리 게시물을 확인하고 관리하는 화면이다.

관리 항목은 다음과 같다.

- 제목
- 작성자
- 날짜
- 장소
- 대표사진
- 추가사진
- 공개 상태

관리자는 부적절한 게시물을 비공개 또는 숨김 처리할 수 있다.

### 회원 관리

회원 목록과 회원 활동을 확인하는 화면이다.

관리 항목은 다음과 같다.

- 이름
- 연락처
- 지역
- 이메일
- 생년월일
- 가입일
- 작성글 수
- 예약 수
- 교육신청 수

1차 개발에서는 회원 차단이나 등급 기능은 넣지 않고 목록 확인과 상세 확인 중심으로 구성한다.

### 통계

1차 개발에서는 단순 요약 통계만 제공한다.

표시 항목은 다음과 같다.

- 월별 프로그램 신청자 수
- 방문자 통계
- 월별 공간예약 수
- 문의 처리 현황

공간별 예약 횟수 통계는 1차 개발에서는 제외한다.

### 관리자 기록

관리자가 중요한 데이터를 변경했을 때 최소 기록을 남긴다.

기록 대상은 다음과 같다.

- 공간예약 승인, 반려, 취소
- 문의 답변 작성
- 게시글 비공개 처리
- 게시글 숨김 처리
- 갤러리 숨김 처리

기록 항목은 다음과 같다.

- 작업자
- 작업일시
- 작업 종류
- 대상
- 변경 전 상태
- 변경 후 상태

## 4. 1차 개발 우선순위

### 1단계: 관리자 로그인과 접근 제어

- Supabase Auth 기반 로그인 연결
- 관리자 권한 확인
- 관리자만 관리자 페이지 접근 가능
- 일반 이용자는 관리자 페이지 접근 차단

### 2단계: 관리자 대시보드

- 신규 공간예약 수
- 신규 교육신청 수
- 미답변 문의 수
- 오늘 예약
- 이번 주 예약
- 최근 게시글

### 3단계: 공간예약 관리

- 공간예약 목록
- 공간예약 상세
- 승인, 반려, 취소 상태 변경
- 상태 변경 기록 저장

### 4단계: 교육신청 관리

- 프로그램별 신청자 목록
- 신청완료, 대기, 승인, 취소 상태 관리
- 정원 초과 시 대기 상태 처리

### 5단계: 문의 / 제안 관리

- 문의 목록
- 문의 상세
- 답변 작성
- 처리상태 변경
- 회원 마이페이지 또는 비회원 조회 화면과 연결

### 6단계: 게시판과 갤러리 관리

- 공지사항 등록, 수정, 숨김 처리
- 마을 이야기 관리
- 갤러리 관리
- 첨부파일과 이미지 업로드

### 7단계: 회원 관리와 통계

- 회원 목록
- 회원 상세
- 월별 프로그램 신청자 수
- 방문자 통계

## 5. 1차 개발에서 제외할 항목

다음 기능은 2차 개발 또는 운영 안정화 이후로 미룬다.

- 문자 알림
- 카카오 알림톡
- 이메일 자동 알림
- 관리자 권한 세분화
- 출석 관리
- 수료 관리
- 공간별 예약 통계 상세 분석
- 행사별 갤러리 앨범
- 반려 사유 필수 입력

- 회원 정보
